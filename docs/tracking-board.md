## Usage
1. **Assign owners:** each issue must list the primary developer and reviewer before moving columns.
2. **Due dates & milestones:** phases auto-populate milestone start/end dates; adjust only if sprint cadence changes.
3. **Risk log:** capture blocking risks in issue comments and cross-link in `docs/gitlab/issues-triage.md`.
4. **Feature flags:** log new flags in `docs/feature-flags.md` and reference the issue from the board card.
5. **Automation:** rerun `npm run gitlab:boards` after updating phase headings or adding new phase labels.
